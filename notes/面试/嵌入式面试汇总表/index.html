    一、存储与启动（5题）¶

    1. Flash Memory 是什么？¶

    主回答

    Flash Memory（闪存）是非易失性存储器，断电不丢数据。常见两种： - NOR Flash：适合代码存储、支持XIP（eXecute In Place）随机读 - NAND Flash：容量大、适合批量存储、常用于SSD/eMMC

    嵌入式中用于存储 Bootloader、应用程序、参数等。

    追问1：NOR vs NAND 怎么选？
    答：需要直接执行代码/随机读多→NOR；大容量/顺序读写→NAND（配文件系统/坏块管理）。SSD用NAND。

    追问2：Flash与RAM/ROM的区别？
    答：RAM易失性，掉电丢数据；ROM只读难编程；Flash非易失可多次擦写，兼两者优点。

    2. Bootloader 是什么？¶

    主回答

    Bootloader是上电后执行的第一段程序，流程： 

    上电 → CPU从固定地址取指 → Bootloader运行
→ 初始化时钟/DDR/串口/存储
→ 加载应用到RAM → 跳转执行应用
    通常还带固件升级功能（串口/USB/OTA）。

    追问1：升级如何防止变砖？
    答：双分区A/B切换、升级包CRC/签名校验、断点续传、回滚机制。

    追问2：Bootloader占多少空间？
    答：一般16-128KB；大小取决于功能复杂度和硬件初始化量。

    3. 裸机编程是什么？¶

    主回答

    Bare Metal（裸机）是无操作系统，程序直接跑硬件上。 - 结构：main() → while(1) 主循环 + 中断 - 优点：简单、资源占用小、确定性强 - 缺点：复杂度上升难维护、多任务支持差

    追问1：复杂逻辑怎么组织？
    答：有限状态机(FSM)、事件队列、定时器驱动任务切片；ISR只做"置标志/搬数据"，复杂处理放主循环。

    追问2：什么时候用裸机，什么时候用RTOS？
    答：简单单任务、资源极限→裸机；多任务、实时要求高→RTOS。

    4. GPIO / Timer / PWM 分别做什么？¶

    主回答

    • GPIO：通用输入输出，按键/LED常见应用
    • Timer：计数/定时/捕获比较、触发中断、PWM基础
    • PWM：脉宽调制，通过占空比输出等效模拟量（电机/LED/舵机）

    追问1：PWM频率和分辨率如何权衡？
    答：频率高→噪声小但分辨率可能下降；分辨率由计数范围决定。按负载选频率，尽量提高ARR/PSC范围。

    追问2：舵机为何用50Hz？
    答：脉宽1-2ms映射角度，周期20ms（50Hz）是业界标准，便于同步控制。

    5. 中断控制器（NVIC）是什么？¶

    主回答

    NVIC（Nested Vectored Interrupt Controller）用于ARM Cortex-M系列： - 功能：中断使能、优先级管理、向量跳转、中断嵌套 - 抢占优先级 vs 响应优先级：前者决定能否抢占，后者决定同级顺序

    追问1：中断里能做复杂计算吗？
    答：不建议。ISR原则是短小快速，只做"读寄存器/清标志/置事件"，复杂处理放任务。

    追问2：优先级怎么设置？
    答：硬实时事件高优先级，中断嵌套层数不超过3-4层避免栈爆炸。

    二、单片机基础（8题）¶

    6. 单片机中断基本流程？¶

    主回答

    中断触发 → 保存现场(PC/LR/寄存器)
→ 进入ISR → 处理事件+清中断标志
→ 恢复现场 → 返回主程序
    ISR是被动触发，实时性最强。

    追问1：ISR里最忌讳什么？
    答：长时间阻塞（printf、sleep、等锁），会破坏实时性。用信号量/队列通知任务处理。

    追问2：中断嵌套和优先级反转关系？
    答：嵌套指中断被中断打断；优先级反转是低优先级任务阻塞高优先的情况（RTOS概念）。

    7. 中断向量表是什么？¶

    主回答

    向量表保存各中断/异常入口地址，CPU根据中断号查表跳转ISR。 - 位置：Flash起始地址 - 结构：[MSP, Reset_Handler, NMI_Handler, HardFault_Handler, ...]

    追问1：向量表能重定位吗？
    答：支持，通过VTOR（Vector Table Offset Register）；用于Bootloader/APP分区切换。

    追问2：如何手动修改向量表？
    答：直接定义数组或使用编译器属性（__attribute__），指向各ISR函数。

    8. 中断与轮询区别？¶

    主回答

    • 轮询：主循环不断检查状态，占用CPU、延迟高、浪费功耗
    • 中断：事件驱动，效率高、响应快、适合实时场景

    缺点：中断频繁可能导致系统不稳定。

    追问1：什么时候反而用轮询？
    答：事件极频繁且处理时间短；启动自检读状态；避免中断抖动。

    追问2：按键如何消抖？
    答：中断里只置标志，主循环/定时器延时10-20ms确认；或定时器100Hz采样滤波。

    9. GPIO 开漏输出是什么？¶

    主回答

    开漏输出只能拉低不能主动拉高，需要外部上拉电阻形成高电平。 - 优点：可驱动多个输出线"线与"、支持不同电压 - 应用：I2C、IIC、故障诊断输出

    追问1：为什么I2C要用开漏？
    答：多个从设备共享SDA/SCL，开漏实现"线与"逻辑，任意设备拉低就低。

    追问2：上拉电阻多大合适？
    答：一般10kΩ，需平衡驱动能力与功耗；高速通信可用更小值。

    10. ADC 关键指标有哪些？¶

    主回答

    • 分辨率：如12bit可表示4096个等级
    • 采样率：每秒采样次数（kS/s或MS/s）
    • 精度：转换结果误差
    • 通道数：可接传感器个数
    • 转换时间：单次转换耗时

    追问1：怎么提高测量精度？
    答：增加采样次数求平均、硬件配RC滤波、校准偏移和增益系数。

    11. DMA 是什么？¶

    主回答

    DMA（Direct Memory Access）在外设和内存间直接传输数据，无需CPU参与。 - 优点：CPU负担小、传输快、功耗低 - 应用：ADC连续采样、UART大数据接收、Flash编程

    追问1：DMA+中断怎么组织？
    答：外设触发DMA搬数据到buffer，DMA完成/半满时中断通知，ISR只发信号给任务处理。

    12. 看门狗定时器（Watchdog）怎么用？¶

    主回答

    看门狗是防死机装置：倒计时定时器，程序需定期"喂狗"（重载计数器），否则复位系统。 - 适用场景：长时间运行不能人工复位的设备 - 喂狗周期：通常设为超时时间的1/3到1/2

    追问1：喂狗周期多少合理？
    答：若看门狗设1秒，喂狗周期应<500ms，留出余量防止误触发。

    13. 低功耗模式有哪些？¶

    主回答

    • Sleep：CPU停止，外设继续，唤醒快
    • Stop：大部分时钟关闭，功耗更低，外设可保留
    • Standby：几乎全关，只保留最小唤醒电路，功耗最低

    追问1：什么时候选哪种模式？
    答：短暂空闲→Sleep；长时间空闲/低功耗设备→Stop/Standby；按实际休眠时间权衡唤醒延迟。

    三、通信协议（7题）¶

    14. UART 是什么？¶

    主回答

    UART（Universal Asynchronous Receiver/Transmitter）异步串口通信： - 线：TX、RX（有时+RTS/CTS硬件流控） - 特点：简单、低速、常用于调试和GPS - 波特率：常见9600/115200，MCU需和外设一致

    追问1：如何支持硬件流控？
    答：添加RTS/CTS线；发送方可根据CTS拉低/拉高调整发送速率。

    追问2：串口收包不丢数据怎么做？
    答：用中断或DMA+环形缓冲，按帧头/长度/校验解析；ISR不做重逻辑。

    15. I2C 原理与特点？¶

    主回答

    I2C（Inter-Integrated Circuit）两线同步总线： - 线：SCL（时钟）、SDA（数据），开漏输出需上拉 - 特点：多主多从、软件灵活、集成简单 - 速率：100kHz（标准）、400kHz（快速）、1MHz（高速）

    追问1：I2C为什么要上拉电阻？
    答：开漏输出，上拉到高电平；多设备线与共享。

    追问2：同步和异步区别？
    答：同步有时钟信号同步（SPI/I2C）；异步依赖波特率计时（UART）。

    16. SPI 全双工通信？¶

    主回答

    SPI（Serial Peripheral Interface）高速同步全双工： - 线：SCLK（时钟）、MOSI（主→从）、MISO（从→主）、CS（片选） - 特点：速度快、协议简单、支持全双工 - 应用：Flash、显示屏、高速ADC

    追问1：为什么需要CS片选？
    答：总线可挂多个从设备，CS选择目标并界定传输边界。

    追问2：多从设备怎么接？
    答：每从一条CS线；MOSI/MISO/SCLK并联；主机按序拉低对应CS通信。 Chip Select，片选信号，用于选择SPI总线上的从设备。

    17. CAN 总线优势？¶

    的地位 主回答

    CAN（Controller Area Network）多主差分高可靠总线： - 特点：自仲裁、抗干扰强、错误检测、广泛用于汽车/工业 - 速率：常见500kbps、1Mbps - 帧：标准帧(11位ID)、扩展帧(29位ID)

    追问1：ID低优先级高怎么理解？
    答：仲裁时ID小的优先级高；同时发送时ID小的赢得总线。

    追问2：CAN错误如何处理？
    答：错误主动/被动模式、自动重传、帧检验校验。

    18. I2S 用于什么？¶

    主回答

    I2S（Inter-IC Sound）音频数据传输协议： - 线：BCLK（位时钟）、LRCLK（声道时钟）、SD（数据） - 应用：连接音频Codec、DAC/ADC - 格式：Philips I2S、Left-Justified、Right-Justified

    追问1：为什么需要LRCLK？
    答：区分左右声道，LRCLK决定数据属于哪个声道。

    19. 传感器通信协议怎么描述？¶

    主回答模板

    我从四点说： 1. 协议：SPI/I2C/UART/CAN 2. 速率：如I2C 400kHz、SPI 8MHz、UART 115200 3. 帧格式：帧头+ID+长度+负载+校验(CRC) 4. 可靠性：超时重传、环形缓冲、DMA、中断策略

    追问1：数据偶尔错误怎么办？
    答：加CRC校验、超时重传、软件滤波去毛刺；检查硬件：线长、上拉、终端电阻。

    20. PWM 在哪些场景用？¶

    主回答

    • 电机调速：占空比→速度
    • LED调光：占空比→亮度
    • 舵机控制：脉宽→角度（通常50Hz, 1-2ms）
    • 蜂鸣器：频率→音调

    追问1：多组PWM同步如何实现？
    答：同一定时器多通道或软件同步机制。

    四、RTOS与并发（20题）¶

    21. RTOS vs 裸机？¶

    主回答

    • 裸机：while(1)+中断，简单但难维护、多任务支持差
    • RTOS：多任务、优先级调度、同步互斥、可维护强但资源占用大

    选择：简单→裸机；复杂实时→RTOS

    追问：RTOS和Windows/Linux根本区别？
    答：RTOS保证实时性（任务必须deadline前完成）；通用OS只尽量快。

    22. RTOS 调度器怎么工作？¶

    主回答

    调度器按优先级从就绪队列选择最高优先级任务执行： - 抢占式：高优先级可打断低优先级（实时性好） - 协作式：任务主动让出CPU（简单但实时性弱）

    任务状态：Running → Ready → Blocked → Suspended

    追问1：如何避免优先级反转？
    答：优先级继承或天花板机制。

    追问2：定时器中断也通过调度吗？
    答：定时器中断触发调度器重新选择就绪任务。

    23. 优先级反转是什么？¶

    主回答

    低优先级任务持有资源，高优先级任务等待，中优先级又一直运行，导致高优先被拖延。

    解决：优先级继承（低优先临时继承高优先级直到释放资源）或优先级天花板。

    追问：如何检测优先级反转？
    答：通过日志/trace分析任务调度顺序，观察高优先任务被延迟时长超预期。

    24. 任务间通信方式？¶

    主回答

    • 消息队列：任意数据传递
    • 信号量：事件同步、资源计数
    • 互斥锁：保护共享资源
    • 事件标志组：多事件组合同步
    • 条件变量：线程等待条件

    追问：哪个用于生产者-消费者？
    答：消息队列最直接；或信号量+手动缓冲。

    25. 互斥锁 vs 信号量？¶

    主回答

    • Mutex：互斥，有"所有权"概念，通常初值1，进入临界区lock，离开unlock
    • Semaphore：计数，用于资源控制或事件通知，初值可为N

    用途：保护共享变量→Mutex；事件同步→Semaphore；资源池→Semaphore

    追问：Mutex死锁怎么避免？
    答：固定加锁顺序、超时机制、trylock。

    26. 任务栈管理要注意什么？¶

    主回答

    • 栈大小：按局部变量+调用深度估计，留余量
    • 栈溢出检测：FreeRTOS有uxTaskGetStackHighWaterMark()
    • 栈初值：通常填0xA5或0xCC便于识别
    • 位置：通常在SRAM末尾

    追问：栈溢出有什么后果？
    答：覆盖其他数据、系统崩溃、难以定位的bug。

    27. FreeRTOS 核心概念？¶

    主回答

    • Task：独立执行流，有栈、优先级、状态
    • Scheduler：根据优先级调度就绪任务
    • Tick：系统时钟中断，通常1ms一次
    • Context Switch：任务切换，保存/恢复上下文

    追问：怎么检测任务CPU占用率？
    答：FreeRTOS可配置运行时统计，定时器中断计累计运行时间。

    28. 中断嵌套和优先级？¶

    主回答

    • 中断嵌套：高优先级中断打断低优先级ISR执行
    • 避免过度嵌套：设置合理优先级分层、控制ISR时长
    • 系统稳定：通常不超过3-4层嵌套

    追问：ISR里能调用FreeRTOS API吗？
    答：只能调用带"FromISR"后缀的函数（如xQueueSendFromISR）。

    29. 死锁原因与避免？¶

    主回答

    四个必要条件（破坏任一可避免）： 1. 互斥：资源独占 2. 占有且等待：持有锁再等新锁 3. 不可剥夺：无法强制释放 4. 循环等待：任务循环等待

    避免：固定加锁顺序、超时、trylock、拆分粒度

    追问：哪个条件最好破坏？
    答：循环等待和占有且等待最实际；通过顺序和timout可同时破坏两个。

    30. 信号量二值 vs 计数？¶

    主回答

    • 二值信号量：初值1，只能0/1，用于互斥或简单事件
    • 计数信号量：初值N，用于资源池或多个事件累计

    追问：资源池怎么用信号量？
    答：初值=资源数，获取时-1，释放时+1；0时等待表示资源耗尽。

    31. 条件变量适用什么场景？¶

    主回答

    生产者-消费者模式：消费者等待条件（队列非空），生产者发送信号唤醒。 - 优点：避免忙等或轮询 - 配套：通常和Mutex搭配

    追问：Condition Variable vs Semaphore用哪个？
    答：CV更灵活但复杂；Semaphore简单直接，多数场景够用。

    32. 消息队列原理？¶

    主回答

    环形缓冲结构，生产者写入消息，消费者读取： - 满时生产者可阻塞或返回，空时消费者阻塞等待 - 通常用于任务间数据传递和事件通知

    追问：队列溢出怎么处理？
    答：可选覆盖旧数据、阻塞发送方、丢弃新数据。

    33. Linux 进程vs线程？¶

    主回答

    • 进程：资源分配单位，独立地址空间，切换开销大
    • 线程：调度单位，共享地址空间，切换快但需同步

    安全性：进程隔离强；线程共享内存需谨慎。

    追问：什么时候用进程？
    答：需要隔离、互不影响→进程；任务分工明确但资源共享→线程。

    34. 线程同步工具？¶

    主回答

    • Mutex：互斥锁，保护临界区
    • Condition Variable：条件等待
    • Semaphore：计数同步
    • RWLock：读写锁，多读单写
    • Barrier：同步栅栏

    追问：读写锁什么时候用？
    答：读频繁写稀少，多个读者并行不冲突。

    35. 死锁检测与恢复？¶

    主回答

    • 检测：系统记录资源分配图，检测环路
    • 恢复：终止部分进程、强制释放资源

    实际多采用预防（银行家算法、资源有序分配）而非检测恢复。

    五、C/C++语言（15题）¶

    36. 指针和数组区别？¶

    主回答

    • 指针：变量存储地址，可改指向、可为空、sizeof=指针大小
    • 数组：连续内存块，固定大小、数组名不可改、sizeof=整体大小

    数组名在表达式中退化为指针，但本质不同。

    追问1：函数参数里数组会退化吗？
    答：会退化为指针，因此sizeof(arr)在函数里得指针大小，需另传数组长度。

    追问2：函数指针怎么用？
    答：int (*func_ptr)(int, int) = &add;用于回调函数、驱动接口。

    37. const 和 volatile？¶

    主回答

    • const：编译期只读，放Flash节省RAM
    • volatile：禁优化，每次都从内存读写；用于硬件寄存器、中断变量、多线程

    两者可组合：const volatile int reg = ...;

    追问1：const int p vs int * const p？
    答*：前者数据不可改；后者指针不可改。

    追问2：为什么中断变量要volatile？
    答：中断改变变量，编译器可能优化掉读写导致看不到最新值。

    38. 堆和栈区别？¶

    主回答

    属性 栈 堆
    管理 自动 手动malloc/free
    大小 小（KB） 大（MB）
    速度 快 慢
    易失性 函数返回失效 手动释放
    风险 溢出崩溃 泄漏、碎片

    嵌入式优先用栈，避免malloc。

    追问：栈溢出症状？
    答：覆盖其他变量、异常跳转、系统崩溃。

    39. 内存泄漏怎么检测？¶

    主回答

    • 静态分析：代码审查找malloc无对应free
    • 运行时工具：valgrind、asan(Address Sanitizer)
    • 嵌入式方案：内存池统计、日志追踪

    预防：尽量用栈/静态、RAII模式(C++)、内存池。

    追问：为什么嵌入式不推荐malloc？
    答：碎片化、实时性不确定、资源紧张、不利于功耗管理。

    40. 位操作常用技巧？¶

    主回答

    // 置第n位为1
x |= (1 << n);

// 清第n位
x &= ~(1 << n);

// 翻转第n位
x ^= (1 << n);

// 检查第n位
if (x & (1 << n)) { ... }

    追问：如何一次设置多位？
    答：x |= 0b1111000; 或 x |= (0xF << 4);

    41. 大端小端？¶

    主回答

    • 大端(Big Endian)：低地址存高字节，网络字节序
    • 小端(Little Endian)：低地址存低字节，大多数MCU/PC

    STM32小端；网络通信需htonl/ntohl转换。

    追问：怎么判断机器字节序？
    答：

    int x = 0x12345678;
if (*(char*)&x == 0x78) printf("小端\n");

    42. 结构体对齐与padding？¶

    主回答

    编译器自动插入填充字节对齐，提高访问效率。 

    struct Foo {
  char a;    // offset 0
  // 3字节填充
  int b;     // offset 4
  char c;    // offset 8
  // 7字节填充
}; // sizeof = 16字节

    通信需要：#pragma pack(1) 或 __attribute__((packed))

    追问：union节省内存吗？
    答：是，成员共用内存，大小=最大成员；用于协议解析。

    43. C 和 C++ 在嵌入式选哪个？¶

    主回答

    • C：贴近硬件、代码紧凑、生态成熟、驱动首选
    • C++：OOP、模板、可维护好、业务复杂推荐

    实际：底层驱动用C，应用逻辑用C++；要控制异常、RTTI、STL开销。

    追问：虚函数开销大吗？
    答：有虚表查询开销，但相对可控；关键路径避免或确保频率不极高。

    44. 宏 vs inline函数？¶

    主回答

    • 宏：预处理文本替换，无类型检查，可能有副作用
    • inline：编译期内联，有类型检查，更安全

    用宏做常量、条件编译；函数逻辑用inline。

    追问：宏怎么写才安全？
    答：参数加括号、整体加括号、避免副作用。

    45. strcpy 溢出防护？¶

    主回答

    // 不安全
strcpy(dest, src);

// 安全版本
strncpy(dest, src, sizeof(dest)-1);
dest[sizeof(dest)-1] = '\0';
    或用strlcpy（若可用）、或手动计算长度。

    追问：为什么要-1？
    答：预留\0位置，确保字符串正常结尾。

    46. 递归注意事项？¶

    主回答

    • 必须有终止条件，否则无限递归
    • 栈消耗：每次调用压栈，嵌入式栈小要谨慎
    • 替代方案：循环+显式栈/队列

    递归深度通常<10为安全。

    追问：怎样用迭代替代递归？
    答：斐波那契、树遍历等可用显式栈DFS替代。

    47. 命名空间和static？¶

    主回答

    • namespace：组织符号、防冲突（C++）
    • static：文件作用域（C）或类成员（C++）
    • 外部链接：全局符号其他文件可见
    • 内部链接：static限制本文件

    追问：为什么头文件不写using namespace std?
    答：会污染全局命名空间，导致冲突。

    48. 类的封装和继承？¶

    主回答

    • 封装：private/public/protected隐藏实现
    • 继承：代码复用、多态基础
    • 多态：虚函数实现运行时多态

    嵌入式尽量用组合而非深继承。

    追问：虚函数表怎么工作？
    答：每个类有虚表指针，指向虚函数地址数组；调用时动态查表跳转。

    49. 模板和泛型？¶

    主回答

    模板在编译期实例化生成具体代码，适合容器/算法库。

    缺点：编译复杂、二进制膨胀；嵌入式谨慎用。

    追问：模板代码会膨胀吗？
    答：会，每个类型实例化一份代码。控制模板层数和实例数。

    50. 异常处理开销？¶

    主回答

    C++异常有性能开销（栈展开、类型识别）。

    嵌入式典型做法： - 关闭异常（-fno-exceptions） - 用返回值/错误码而非throw - 关键路径避免异常

    六、嵌入式Linux（9题）¶

    51. Linux 启动流程？¶

    主回答

    1. Bootloader(U-Boot)
   ├─ 初始化硬件、时钟、DDR
   └─ 加载内核到内存

2. Kernel启动
   ├─ 初始化MMU、内存管理
   ├─ 加载设备驱动
   └─ 启动第一个用户进程init

3. Init进程
   ├─ 挂载文件系统
   ├─ 启动系统服务
   └─ 系统进入正常运行

    追问：Kernel代替Bootloader跳转吗？
    答：是；Bootloader加载kernel到RAM，然后跳转到kernel_entry。

    52. 进程 vs 线程详解？¶

    主回答

    比较 进程 线程
    地址空间 独立 共享
    切换开销 大 小
    隔离 强 弱
    创建速度 慢 快
    通信 IPC复杂 直接共享内存

    追问：什么时候多进程比多线程好？
    答：需要强隔离与独立性；一个进程奔溃不影响其他。

    53. IPC 通信方式（进程间）？¶

    主回答

    • 管道：单向、简单、用于parent-child
    • 消息队列：有结构、可多进程
    • 共享内存：最快、需同步机制
    • 信号量：计数同步
    • Socket：网络通信、本地IPC
    • 信号：异步通知

    追问：管道怎么实现双向通信？
    答：两条管道，或全双工socket。

    54. 系统调用是什么？¶

    主回答

    用户进程请求内核服务的接口，从用户态切到内核态： 

    用户程序 → system call(触发中断/陷入)
→ 内核处理 → 返回用户态

    常见：open/read/write/fork/exec/socket等。

    追问：用户态和内核态为什么分离？
    答：隔离与安全，防止用户程序破坏系统。

    55. Linux 常用命令？¶

    主回答

    进程：ps/top/kill/pgrep
    网络：ping/ip/ss/netstat/tcpdump
    文件：ls/cd/sed/grep/find/awk
    性能：free/df/iostat/vmstat
    日志：dmesg/journalctl/tail/less

    追问：怎么看某端口占用？
    答：ss -lntp | grep :端口号 或 netstat -lntp。

    56. TCP 三次握手和四次挥手？¶

    主回答

    三次握手：SYN → SYN+ACK → ACK，建立连接
    四次挥手：FIN → ACK → FIN → ACK，关闭连接

    挥手四次因为TCP全双工，双方各关闭发送方向。

    追问：TIME_WAIT持续多久？
    答：2MSL（Maximum Segment Lifetime），防止旧报文影响新连接。

    57. select/poll/epoll 区别？¶

    主回答

    • select：轮询所有fd，<=1024个，O(n)
    • poll：不限fd数，仍O(n)轮询
    • epoll：事件驱动，O(1)查询，适合高并发

    连接少用select/poll；连接多用epoll。

    追问：epoll的LT和ET？
    答：LT(水平)持续通知有数据；ET(边沿)只通知状态变化边界。

    58. 非阻塞IO和Reactor模型？¶

    主回答

    非阻塞：立刻返回，没数据返回EAGAIN
    配合epoll形成Reactor模式： 

    Main loop:
  events = epoll_wait(...)
  for each event:
    if readable: read_callback()
    if writable: write_callback()

    追问：什么时候用Reactor vs Proactor？
    答：Reactor事件通知就绪，应用读写；Proactor操作系统代为读写。Linux多Reactor。

    59. 网络字节序转换？¶

    主回答

    网络字节序通常大端。转换函数： - htonl/htons：主机→网络(32/16位) - ntohl/ntohs：网络→主机

    STM32小端，网络通信要转。

    追问：怎样写字节序无关的代码？
    答：收数据时用转换函数，或自己定义bswap_32等。

    七、网络编程（13题）¶

    60. TCP 服务器五步建立？¶

    主回答

    1. socket()     // 创建套接字
2. bind()       // 绑定IP+端口
3. listen()     // 进入监听
4. accept()     // 接收连接，返回已连接socket
5. recv/send()  // 收发数据，close()关闭

    追问1：listen和accept区别？
    答：listen把socket变监听态；accept从监听队列取连接，返回新socket专供该客户端。

    追问2：并发服务器怎么做？
    答：多进程/多线程处理每个连接，或epoll事件驱动。

    61. TCP vs UDP 选择？¶

    主回答

    特性 TCP UDP
    连接 虚连接 无连接
    可靠 有重传 不保证
    速度 较慢 快
    适用 数据完整优先 实时延迟优先

    实时音视频/IoT→UDP；文件传输/邮件→TCP。

    追问：UDP怎么保证可靠？
    答：应用层加序号、ACK、重传、超时判定。

    62. TCP 粘包/拆包？¶

    主回答

    TCP无消息边界，多次send可能被合并或拆分。

    解决方案： 1. 固定长度：每帧固定大小 2. 分隔符：帧尾加特殊字符 3. 长度字段(推荐)：Header | Length | Payload | CRC

    追问：UDP会粘包吗？
    答：不会，UDP保留报文边界；但可能丢包、乱序。

    63. Socket 连接建立过程？¶

    主回答

    客户端： 

    socket() → connect()  // SYN
          ← SYN+ACK
          → ACK
    服务器accept()返回，获得已连接socket。

    追问：connect超时多久？
    答：通常75秒或系统默认；可用setsockopt设SO_RCVTIMEO/SO_SNDTIMEO。

    64. Socket 地址结构？¶

    主回答

    struct sockaddr_in {
  sa_family_t sin_family;    // AF_INET
  uint16_t sin_port;         // 端口(网络字节序)
  struct in_addr sin_addr;   // IP
  char sin_zero[8];          // 填充
};

    追问：为什么要填充？
    答：和struct sockaddr保持大小一致。

    65. Listen 的 backlog 参数？¶

    主回答

    backlog表示监听队列长度，超过客户端连接被拒绝。

    实际队列长度 = min(backlog, /proc/sys/net/core/somaxconn)

    追问：连接过多导什么问题？
    答：新连接被拒绝、客户端收RST。

    66. TCP KeepAlive 机制？¶

    主回答

    长连接空闲时定期发送心跳，检测连接是否断开： - SO_KEEPALIVE套接字选项启用 - TCP_KEEPIDLE/INTVL/CNT调整参数

    追问：应用层心跳和TCP KeepAlive怎么选？
    答：KeepAlive更底层通用；应用层心跳更可控，可附带业务数据。

    67. UDP 服务器？¶

    主回答

    1. socket(AF_INET, SOCK_DGRAM)
2. bind()
3. recvfrom()  // 接收数据包+源地址
4. sendto()    // 发送到源地址
无需listen/accept

    追问：怎么给UDP加可靠性？
    答：应用层序号确认、超时重传、丢包检测。

    68. Epoll 高效的原因？¶

    主回答

    • 基于事件：内核通知就绪fd，不用轮询所有fd
    • 红黑树管理：高效查询和删除
    • 就绪列表：O(1)获取就绪事件
    • mmap：内核缓冲区和用户空间共享，无拷贝

    追问：EPollET怎么用？
    答：必须非阻塞且一次读空/写尽，避免遗漏事件。

    69. 阻塞、非阻塞、异步的区别？¶

    主回答

    • 阻塞：调用不返回直到操作完成
    • 非阻塞：立刻返回，通常需轮询或事件通知
    • 异步：发起后立刻返回，通过回调/信号/事件通知结果

    select/epoll属非阻塞+事件驱动；aio属异步。

    追问：Linux用异步IO吗？
    答：libaio可用，但复杂；多数用epoll非阻塞。

    70. Socket 超时和错误处理？¶

    主回答

    // 设置接收超时
struct timeval tv;
tv.tv_sec = 5;
setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, &tv, sizeof(tv));

// 检查错误
if (recv(...) < 0) {
  if (errno == EAGAIN) {
    // 超时
  } else {
    // 其他错误
  }
}

    追问：连接中断怎么快速检测？
    答：send返回-1 EPIPE；或用select、SO_KEEPALIVE。

    71. 控制连接 vs 数据连接（FTP）？¶

    主回答

    FTP有两条连接： - 控制连接：端口21，传命令 - 数据连接：端口20(被动模式其他)，传文件

    主动/被动模式：谁发起数据连接。

    追问：为什么要分离？
    答：控制继续发命令，数据独立传输更灵活。

    72. HTTP Keep-Alive ?¶

    主回答

    HTTP/1.0 默认每请求一个连接、HTTP/1.1 默认Keep-Alive复用连接

    优势：减少连接建立开销、服务器资源利用率更高

    追问：什么时候关闭连接？
    答：超时、客户端/服务器显式关闭、连接数限制。

    八、硬件设计（8题）¶

    73. PCB 设计基础？¶

    主回答

    • 布线：关键信号单独布线、避免过孔干扰
    • 地和电源：大面积铺地、电源多点供电
    • 阻抗控制：高速信号设计特征阻抗
    • 信号完整性：端接电阻、过孔最小化、串终或并终

    追问：什么时候用串终传电阻？
    答：发送端速率高、传输线长、源阻抗低；并终配50Ω。

    74. 上拉下拉电阻作用？¶

    主回答

    • 上拉：拉到VCC，默认高电平，常用于按键/I2C
    • 下拉：拉到GND，默认低电平
    • 作用：避免引脚悬空、支持线与逻辑

    电阻值通常10k，高速可更小。

    追问：开漏为什么要上拉？
    答：开漏输出无法主动拉高，上拉电阻和外部设备"线与"实现共享。

    75. 调试工具：万用表、示波器、逻辑分析仪？¶

    主回答

    • 万用表：测电压/电流/电阻，DC/AC
    • 示波器：观察波形，触发捕获、FFT频谱分析
    • 逻辑分析仪：数字信号捕获，协议解析(SPI/I2C/UART)

    配合使用定位硬件问题。

    追问：逻辑分析仪能测模拟信号吗？
    答：不能，只捕获高低电平；示波器能看波形。

    76. 电源设计：LDO vs DCDC？¶

    主回答

    • LDO：简单、低纹波、効率低(输入-输出压差损耗)、用于辅助电源
    • DCDC：高效、发热小、电磁干扰、用于主电源

    一般DCDC+LDO级联。

    追问：怎么选LDO参数？
    答：输入输出电压、最大电流、纹波要求、静态功耗。

    77. EMI/EMC 设计？¶

    主回答

    EMI(电磁干扰)/EMC(电磁兼容)： - 抑制源：滤波、隔离、屏蔽 - 加强抗扰度：绕线、磁珠、TVS保护、地平面 - 规范：CE/FCC认证标准

    追问：磁珠作用？
    答：高频滤波，阻抗随频率变化，隔离数字/模拟部分。

    78. 晶体振荡器选择？¶

    主回答

    • 频率精度：±ppm级，越小越准
    • 频率范围：MCU工作频率需匹配
    • 负载容量：电容选择需配匹
    • 温漂：极限温度范围偏差

    通常8MHz或24MHz。

    追问：晶体连接电容多大？
    答：通常10-50pF，根据晶体负载容量标注。

    79. 防护电路：TVS、齐纳二极管？¶

    主回答

    • TVS：快速响应瞬间浪涌，两端保护
    • 齐纳：恒压稳压，单向
    • TVS常用于：GPIO、通信口、电源输入

    选择参数：反向待机电压、浪涌电流。

    追问：ESD防护怎么做？
    答：ESD二极管/芯片、布线避免长走线、大面积地。

    80. 热管理：散热、降频、休眠？¶

    主回答

    • 散热：散热片、风扇、空间设计
    • 降频：温度高time base降频减热
    • 休眠：长期不用切低功耗模式
    • 监测：温度传感器告警

    追问：MCU温度如何监测？
    答：内部温度传感器ADC采集，或外部NTC热敏电阻。

    九、工程实践（8题）¶

    81. 交叉编译是什么？¶

    主回答

    在一种平台上编译代码，在另一种架构平台运行。

    例如：PC(x86) × arm-none-eabi-gcc → 编译 → STM32(ARM)运行

    需要交叉编译工具链、库、头文件。

    追问：怎么配置工程？
    答：CMake/Makefile指定-march、-mthumb；或使用嵌入式IDE自动配置。

    82. HAL层设计目的？¶

    主回答

    Hardware Abstraction Layer 屏蔽硬件差异： - 应用不直接操作寄存器，调用统一接口 - 换芯片：只修改HAL，上层代码不变 - 分工清晰：驱动开发者维护HAL，应用开发者调用

    例：GPIO初始化统一为gpio_init(pin, direction)。

    追问：HAL代码怎么组织？
    答：hal/gpio.h、hal/uart.h等接口；platform/stm32/gpio.c等实现。

    83. OTA升级工程方案？¶

    主回答

    新固件下载 → Flash备用区写入
         ↓
         校验(CRC/SHA)成功?
         ↓YES      ↓NO回滚
         更新启动标志位    保持旧版本
         下次上电: 新程序启动

    双分区/三分区策略保证可靠性。

    追问：升级中掉电怎么办？
    答：校验失败自动回滚到旧版本，或断点续传重新升级。

    84. 内存对齐和padding？¶

    主回答

    数据按特定字节边界存储，提高CPU访问效率。 

    struct X {
  char a;      // offset 0, 1字节
  // padding 3字节(4字节对齐align)
  int b;       // offset 4
}; // total 8字节

    通信/协议需消除padding：#pragma pack(1)。

    追问：大对象怎么省内存？
    答：重排成员（大在前小在后）、压包、或union共用。

    85. 编译优化选项？¶

    主回答

    • -O0：不优化，调试友好
    • -O1/O2/O3：优化级别递增，编译时间变长
    • -Os：优化大小，嵌入式常用
    • -Ofast：忽略IEEE浮点标准，速度最快

    通常发布用-O2或-Os。

    追问：-O会影响调试吗？
    答：会，变量优化掉、指令顺序改变；调试版用-O0。

    86. 链接脚本（Linker Script）？¶

    主回答

    链接脚本定义程序内存布局： 

    MEMORY {
  FLASH (rx)  : ORIGIN = 0x08000000, LENGTH = 256K
  RAM (rwx)   : ORIGIN = 0x20000000, LENGTH = 64K
}

SECTIONS {
  .text : { *(.text*) } > FLASH
  .data : { *(.data*) } > RAM AT > FLASH
}

    追问：怎样把代码放到指定地址？
    答：__attribute__((section(".custom_section"))) + 链接脚本映射。

    87. GDB调试技巧？¶

    主回答

    arm-none-eabi-gdb app.elf
(gdb) target remote :3333   // 连JTAG/SWD调试器
(gdb) load                   // 烧写程序
(gdb) break main             // 设断点
(gdb) continue/next/step     // 执行
(gdb) print var              // 查变量
(gdb) backtrace              // 看堆栈

    追问：怎么远程调试？
    答：OpenOCD开调试服务器，gdb通过网络连接。

    88. 性能分析和优化？¶

    主回答

    • 代码覆盖率：哪些分支未测到
    • 执行时间分析：gprof、perf找热点
    • 内存占用：nm、size工具查符号大小
    • 功耗分析：示波器测电坏功耗、模型计算

    按优先级优化热点代码。

    追问：怎么减少代码体积？
    答：-Os优化、干掉未用函数(--gc-sections)、压缩数据、STL谨慎用。

    十、实战高频场景（5题）¶

    89. 按键消抖实现？¶

    主回答

    // 方案一：延时确认
if (KEY_DOWN) {
  delay_ms(20);     // 延时消抖
  if (KEY_DOWN)     // 再次确认
    handle_key();
}

// 方案二：定时采样(50ms周期)
if (cnt < THRESHOLD) {
  if (KEY_DOWN) cnt++;
  else cnt = 0;
}
if (cnt >= THRESHOLD)
  handle_key();

    追问：硬件消抖用什么元件？
    答：RC滤波电路，时间常数RC>>按键抖动时间(~5ms)。

    90. 环形缓冲（Ring Buffer）实现？¶

    主回答

    typedef struct {
  uint8_t buffer[BUFFER_SIZE];
  uint16_t write_index, read_index, count;
} RingBuffer;

void ring_put(RingBuffer *rb, uint8_t data) {
  rb->buffer[rb->write_index++] = data;
  if (rb->write_index >= BUFFER_SIZE) rb->write_index = 0;
  rb->count++;
}

uint8_t ring_get(RingBuffer *rb) {
  uint8_t data = rb->buffer[rb->read_index++];
  if (rb->read_index >= BUFFER_SIZE) rb->read_index = 0;
  rb->count--;
  return data;
}

    追问：满了怎么办？
    答：覆盖旧数据、丢弃新数据或阻塞；根据应用选择。

    91. PID 控制基础？¶

    主回答

    error = setpoint - current;
P_output = KP * error;
I_output += KI * error;
D_output = KD * (error - prev_error);
output = P_output + I_output + D_output;
    • P：响应速度、超调大
    • I：消除稳态误差（积分）
    • D：抑制超调（微分阻尼）

    应用：电机速度环、PLC温度控制。

    追问：积分积累怎么限制？
    答：积分限幅、条件积分（只在偏差小时积分）、抗积分饱和。

    92. 故障排查方法？¶

    主回答

    系统卡死： 1. 看门狗是否复位？ 2. 串口日志到哪停？ 3. 栈溢出？(uxTaskGetStackHighWaterMark()) 4. 死锁？(gdb看各线程阻塞在哪)

    高CPU：top/htop找病症进程，strace追系统调用，perf profile找热点。

    内存泄漏：valgrind运行，查未释放块。

    追问：没有调试器怎么办？
    答：大量日志打印、LED指示、遥测数据上报、后处理分析日志。

    93. 通信协议设计？¶

    主回答模板

    我从四点描述：

    1. 协议层次：物理层(UART/SPI/...)、链路层(帧格式)、应用层(命令格式)
    2. 帧格式： 
      [帧头 1byte] [命令 1byte] [长度 2byte] [数据 N byte] [CRC 2byte]
0xAA          0xXX        Length        Payload      CRC16
    3. 可靠性：
    4. CRC校验检错，重传机制
    5. 超时处理，连续帧号识别重复
    6. 流控：卡发送方速率匹配接收方处理能力
    7. 测试：构造异常包、抖动测试、压力测试

    追问：怎么处理字节流中的帧头？
    答：状态机逐字节解析，匹配帧头进入数据收集状态。

    94. 状态机（FSM）在嵌入式的应用？¶

    主回答

    例如蓝牙配对流程： 

    [未配对] --收到配对请求--> [配对中]
                         |
                   --输入配对码-->
                         |
                         v
                   [校验成功] --YES--> [已配对]
                         |
                        NO
                         v
                   [配对失败] --重试--> [配对中]

    优势：逻辑清晰，易维护和扩展。

    追问：超时怎么处理？
    答：每状态设超时计数器，超时时复位或重试。

    95. 国际化和本地化考虑？¶

    主回答

    • 字符编码：UTF-8通用
    • 时间：格林威治时间(UTC)+local offset
    • 存储：大小端转换、网络字节序
    • 显示：RTL(右到左)语言支持、字体

    实际产品需多语言字符串表、locale支持。

    追问：怎样存储多语言字符串？
    答：字符串ID映射到资源表，根据locale选不同翻译版本。
