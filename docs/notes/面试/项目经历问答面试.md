
# 🟢 一、项目A：温度指示系统（Arduino）

## 1. 讲一下这个项目的整体架构？

**答：**
该系统基于 Arduino Mega2560，通过 I2C 读取温度传感器数据，将温度映射为电机角度，并通过编码器实现闭环控制。整体包括：

* 传感器采集（I2C）
* 数据处理（温度→角度映射）
* 电机控制（PWM）
* 反馈控制（编码器）
* 状态机控制

---

## 2. 为什么使用 I2C？

**答：**
I2C 只需两根线，支持多设备，适合低速传感器通信。但速率较低（一般100kHz~400kHz），不适合高速场景。

---

## 3. 温度到角度是如何映射的？

**答：**
使用线性映射：

```math
angle = k × temperature + b
```

根据温度范围和机械角度范围进行比例换算。

---

## 4. 编码器如何判断方向？

**答：**
通过 A/B 相位差：

* A领先B → 正转
* B领先A → 反转

---

## 5. 状态机控制有哪些状态？

**答：**

* 粗调（快速接近目标）
* 微调（精确定位）
* 稳定（停止）

---

## 6. 如何避免电机抖动？

**答：**

* 引入死区（deadband）
* 使用微调阶段
* 控制PWM精度

---

## 7. PWM频率如何选择？

**答：**

* 太低：电机抖动
* 太高：驱动损耗大
  通常选择几kHz

---

## 8. 如何调试系统不稳定问题？

**答：**

* 检查编码器反馈
* 查看PWM输出
* 串口打印调试
* 分模块测试


✅ 一、什么是增量式编码器（核心定义）

👉 一句话：

增量式编码器是一种用于测量旋转“变化量”的传感器，通过输出脉冲信号来表示转动的位移和方向。

# 🟡 三、项目C：BeagleBone Linux
Raspberry Pi和beagle bone是一类吗？它们都是单板计算机（SBC），但BeagleBone更注重工业应用，提供更多I/O接口和实时性能，而Raspberry Pi更适合教育和娱乐用途，社区更活跃。



第三个项目是基于 BeagleBone 的嵌入式系统开发，运行在 Linux 系统上。我使用 C 语言开发用户态程序，通过 I2C 接口读取 LIS3DH 加速度传感器的数据。

在控制部分，我实现了一个线性滑台的位置控制系统，通过 PWM 控制电机，并结合编码器反馈实现闭环控制。同时使用 pthread 和 mutex 构建了多线程架构，实现数据采集和控制的并行处理。

在控制算法上，我实现了 PID 控制，用于提高系统响应速度和稳定性。

在这个项目里，我没有做多进程，而是做了一个单进程、多线程的结构。整个程序运行在 Linux 用户态，主线程负责状态机和整体流程，另外创建了一个用于人机交互的线程；在控制逻辑上，多个执行单元会共享同一个状态结构体，所以我用 pthread 创建线程，用 mutex 保护共享数据，避免采集线程和控制线程同时读写同一组变量时出现竞争。

具体来说，我定义了一个 status 结构体，里面保存了目标位置、目标速度、实际位置、实际速度以及当前模式，这个结构体在项目文档里就是核心共享数据区。文档里也明确给了 pthread_mutex_t access，用于保护这几个变量。

实现上可以理解成这样：

主线程运行状态机，根据当前模式执行 START / RUN / IDLE / STOP。在 RUN 模式下周期性读取编码器位置，计算误差，执行 PID，然后更新 PWM 输出。

辅助线程负责 IHM，也就是用户命令输入，比如切换模式、读取当前位置、设置目标位置和目标速度。项目说明里也写了，IHM 是由 secondary thread 管理，而主线程负责状态机。

因为这两个线程都会访问同一个 status 结构体，所以我在访问共享变量前先加锁，操作完成后再解锁。比如 IHM 线程在修改 des_pos、des_vel 或 mode 时，会先 pthread_mutex_lock(&status.access)，写完以后 pthread_mutex_unlock(&status.access)；同样，控制线程在读取目标值、更新 act_pos 和 act_vel 时也会这样做。这样可以保证控制计算拿到的是一致的数据，不会一边算 PID、一边目标值又被另一个线程改掉。文档附录也专门说明了 mutex 用来保护共享资源。

如果面试官继续问“并行处理体现在哪”，你可以接着说：

并行处理主要体现在，一边线程负责和用户交互、更新目标指令，另一边线程持续执行状态机和控制算法，这样系统不会因为等待输入而停止控制，也不会因为控制循环阻塞界面响应。

MEMS(Micro-Electro-Mechanical Systems)微电机系统，加速度传感器是一种微型化的传感器，利用微机械结构测量加速度。它通过检测内部质量块的位移来计算加速度，常用于手机、无人机等设备中。

# 🧠 五、项目E：AI Role Mining（Wavestone）

## 30. 项目做了什么？

**答：**
基于Python构建权限数据，使用无监督学习进行Role Mining，识别异常权限。

---
🔵 1. K-Means（最基础🔥）

K-Means 是一种基于距离的聚类算法，把用户按照权限相似性分成 K 个簇，每个簇可以看作一个角色。

👉 核心：

距离（相似度）

需要指定 K

适合结构比较清晰的数据，但对初始 K 比较敏感。

🟢 2. DBSCAN（密度聚类）

DBSCAN 是基于密度的聚类方法，可以自动发现任意形状的簇，并能识别噪声点。

👉 核心：

密度（邻域点数）

不需要指定簇数量

在权限数据中可以识别异常用户或异常权限分配。

🟡 3. NMF（非负矩阵分解）

NMF 是将用户-权限矩阵分解为两个低维非负矩阵，从而提取潜在的角色结构。

👉 核心：

矩阵分解

可解释性强

每个分解出来的“基向量”可以理解为一个角色。

🔴 4. Autoencoder（自编码器）

Autoencoder 是一种神经网络，通过压缩和重建数据，学习数据的低维表示。

👉 核心：

编码（降维）

解码（重建）

可以捕捉复杂的非线性关系，比传统方法更灵活。

🟣 5. GNN（图神经网络）

GNN 是在图结构上进行学习的方法，可以建模用户和权限之间的关系网络。
