简介
这个项目是一个基于嵌入式 Linux 的线性滑台位置控制系统，主要目标是实现滑台的精确位置控制，同时保证系统的稳定性和安全性。

整个系统运行在 BeagleBone 平台上，我主要做的是从底层驱动到控制算法再到系统架构的完整实现。

从整体结构上来看，这个系统可以分为三部分：

第一部分是执行机构，也就是电机。我通过 PWM 信号控制电机的转速，同时用 GPIO 控制电机的转动方向。

第二部分是反馈系统，我使用了增量式编码器，通过 eQEP 模块实时读取位置，并计算当前速度，这样可以形成闭环控制。

第三部分是控制算法，我实现了一个基于 PID 的位置控制器，根据目标位置和当前实际位置之间的误差，动态调整 PWM 输出。

在软件架构上，我采用的是多线程加状态机的设计。

系统里有两个主要线程：

一个是主控制线程，负责运行状态机，包括 START、RUN、IDLE 和 STOP 四种模式；

另一个是 IHM 线程，用来做人机交互，比如输入目标位置、查询当前状态等。

这两个线程之间通过共享结构体进行通信，并且我使用了 pthread的 mutex（互斥锁）来保护共享数据，避免竞态条件。

在核心控制部分，也就是 RUN 模式中，我实现了一个完整的 PID 控制流程。

每个控制周期大约是 20 毫秒，我会先读取当前的位置和速度，然后计算误差：

包括位置误差和速度误差。

然后通过 PID 公式计算控制量，最后把这个控制量转换成 PWM 信号输出给电机。

在实现 PID 的过程中，我还做了一些工程优化：

首先是积分限幅，防止积分项过大导致系统不稳定；

其次是抗积分饱和，当输出达到 PWM 上限时，我会回退积分，避免控制器发散；

另外我还加了一个死区，当误差很小的时候直接停止电机，避免在目标点附近抖动。

除了控制算法，我还实现了一些比较重要的工程功能。

第一个是限位保护，系统左右各有一个限位开关，如果检测到已经到达边界，就会禁止继续往那个方向运动，防止机械撞击。

第二个是 homing，也就是回零流程。

系统启动后会先以低速向左移动，直到触发限位开关，然后把当前位置设为零点，这样就建立了一个物理参考坐标系。

另外在轨迹控制方面，我没有直接让滑台跳到目标位置，而是做了一个简单的轨迹生成。

通过给定速度，按照一个线性函数逐步更新目标位置，这样滑台运动更加平滑，也可以减少冲击和过冲。

最后，我还扩展了一个传感器模块，接入了 LIS331HH 三轴加速度传感器。

通过 I2C 总线读取加速度数据，并在系统中实时更新，这样可以用来监测滑台的运动状态，比如振动或者加速度变化。

总结一下，这个项目的几个核心点是：

我实现了一个完整的闭环控制系统，包括 PWM 驱动、电机控制、编码器反馈和 PID 控制；

在软件上采用了多线程和状态机结构，提高了系统的清晰性和可扩展性；

同时还加入了限位保护、回零流程和轨迹生成，使系统更接近实际工程应用；

另外还扩展了 IMU 传感器，提高了系统的感知能力。
