    温度指示项目设计思路与调试总结¶

    1. 编码器计数的来源¶

    编码器计数并不是由软件“定义”出来的，而是由编码器硬件结构及其输出脉冲数决定。程序通过 Encoder 库对 A/B 相信号进行解码并累积计数。

    经实验测得，手动转动驱动轮一圈，编码器计数变化为 12，因此这个数值被用于后续角度换算与控制逻辑。

    验证时用过如下最小代码：

    void loop() {
  Serial.println(enc.read());
}

    因此，可以通过读取编码器计数来确定驱动轮的旋转角度和位置，从而实现较精确的运动控制。

    2. 最初的功能链路¶

    最初思路非常直接：

    1. 先读取温度数据。
    2. 再把温度数据线性转换为角度。

    后续逐步补上了校准、状态机和细化控制逻辑。

    3. 关键验证步骤¶

    3.1 温度读取验证¶

    确认 I2C 能稳定读取温度值，并且 Arduino 能正确解析为摄氏度数值，为后续角度映射提供可靠输入。

    3.2 编码器验证¶

    确认驱动轮转一圈时 count = 12，同时确认计数方向的正负与物理转动方向一致。

    3.3 校准动作验证¶

    为解决没有绝对零位的问题，设计了以下校准动作：

    • r / l：右转 / 左转一圈
    • r5 / l5：右转 / 左转五圈

    其中，字母表示方向，数字表示转动圈数。

    4. 发现的问题¶

    在单步动作验证阶段，发现单纯使用 PI 控制时，r / l 指令很难稳定、准确地到达目标位置，控制难度较大。

    主要原因包括：

    • 编码器分辨率较低，反馈是离散计数
    • 系统存在明显静摩擦和齿隙
    • PWM 有“能动门槛”，积分项容易累积后突然释放

    这些因素会让 PI 控制更容易振荡和过冲，尤其是在短距离校准动作中更明显。

    5. 控制策略改进¶

    为提高控制精度，引入了 pulse-based integral compensation 的思路，并进一步设计了 nudge 微调模式。

    核心想法是：

    • 误差较大时，用 P 控制快速逼近
    • 误差较小时，用短时 PWM 脉冲推进
    • 每次脉冲后强制断电等待系统停稳，再重新评估误差

    这样做在工程效果上接近积分补偿，但更适合当前这种低分辨率编码器和强非线性机械系统。

    最终目标是：

    • 输入 r / l / r5 / l5 后，计数误差控制在 ±1 以内
    • 并且大部分时候可以落到 0

    6. 状态机设计¶

    为了实现这个思路，需要设计以下状态机：

    • M_IDLE
    • M_COARSE_FAST
    • M_COARSE_SLOW
    • M_NUDGE_OFF
    • M_NUDGE_ON

    含义分别是：

    • M_IDLE：空闲状态，电机关闭
    • M_COARSE_FAST：粗调快速逼近
    • M_COARSE_SLOW：粗调慢速逼近
    • M_NUDGE_OFF：断电等待停稳
    • M_NUDGE_ON：脉冲式微调推进

    状态机会根据当前误差大小在不同状态之间切换，实现从快速响应到高精度定位的平滑过渡。

    7. 设计思路整理¶

    7.1 问题背景与总体目标¶

    本项目旨在设计并实现一种基于电机与编码器的指针式温度显示系统。系统通过温度传感器实时采集环境温度，并驱动电机带动指针旋转，使指针指向与当前温度对应的刻度位置。

    由于所采用的电机与传动机构不存在物理上的绝对零位，同时系统存在静摩擦、齿隙和负载不确定性等非理想因素，因此必须采用闭环控制结构，并设计可靠的人工校准机制，以保证系统在长期运行中的准确性和稳定性。

    7.2 系统功能拆分与总体架构¶

    根据功能需求，可以把系统划分为两个工作层次。

    目标生成层：

    • 根据温度传感器输出生成期望指针角度
    • 根据人工校准指令（r / l / r5 / l5）生成相对位移目标

    位置控制层：

    • 以编码器计数（count）作为唯一反馈量
    • 通过闭环控制驱动电机，使实际位置逼近期望位置

    这样一来，校准动作与温度跟随动作就可以共享同一套位置控制器，仅在“目标如何生成”上有所区别。

    8. 最终效果¶

    通过上述设计，系统最终实现了：

    • r / l / r5 / l5 校准指令后，计数误差稳定控制在 ±1 以内，大多数情况下为 0
    • 同一套状态机与控制逻辑可同时用于：
    • 人工校准（相对位移控制）
    • 温度跟随（绝对位置控制）

    两种模式仅在目标生成方式上不同，而底层位置控制器完全统一，因此整体结构更简洁、也更容易维护。

    9. 小结¶

    最终采用了 P 控制 + nudge 微调 的方式，而不是继续硬套传统 PI 控制。

    本项目把控制重点统一到编码器计数域，并通过状态机实现粗调与微调结合，因此在低分辨率编码器和强非线性机械条件下，仍然实现了稳定、精确且可校准的指针式温度显示系统。
